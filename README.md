# 公募基金托管法规知识库使用说明

## 1. 法规库简介

本法规库面向商业银行开展公募基金托管业务，整理了托管相关的法律法规、监管规则、行业自律规则、交易所和登记结算规则、会计估值与清算规则等资料。

它的用途不是替代法律意见，而是为本地 agent 提供一套可检索、可核验、可维护的法规依据链。回答问题时，应优先使用本地库，再按需联网核验最新性和官方来源。


## 2. 使用指南：让本地 Agent 读取法规库回答问题

将整个法规库文件夹下载到本地后，用支持读取本地文件的 agent 打开本项目目录，并让 agent 按本库规则回答问题。

适合提问的方式：

- `ETF 基金估值适用摆动定价吗？`
- `基金托管人在估值复核中有哪些职责？`
- `公募基金投资同业存单的估值和会计核算依据有哪些？`
- `某条规则是否现行有效？本地库有没有官方来源？`

agent 回答时会根据根目录 `AGENTS.md` 的指示遵循这条工作链路：

1. 先从本地索引快速召回候选法规和条款；
2. 再用元数据台账确认法规名称、效力状态、优先级、官方来源和本地路径；
3. 对业务场景型问题，补查托管业务专题地图，避免遗漏相关规则；
4. 回看文本上下文，并回到原文文件和官方 URL 核验；
5. 回答中区分“原文依据”和“分析判断”；
6. 如果结论依赖但本地库未收录相关规则、附件或配套文件，应登记到 `99_unresolved/unresolved.md`，并说明本地库暂未收录。

agent 不会凭模型记忆回答法规问题，也不要把征求意见稿、修订说明、新闻稿、解读文章、行业手册或操作实务手册当作现行正式规则。

## 3. 维护指南：通过自动化任务定期更新

建议为本法规库配置定期自动化任务，例如每月巡检一次，并在监管规则集中发布、业务发生变化或发现本地库缺口时临时触发更新。

自动化任务的目标不是直接给出未经复核的最终结论，而是完成以下工作：

1. 检查官方来源是否有新增、修订、废止或附件变化；
2. 下载或更新法规原文，并生成可检索文本；
3. 刷新元数据台账、总目录、专题地图、待核验清单和本地搜索索引；
4. 标注仍未确认官方正文、附件或现行状态的项目；
5. 输出本次新增、变更、失效、待核验事项，供人工复核。

每次更新后agent会至少检查并按需同步这些文件：

- `README.md`
- `00_入口与索引/总目录.html`
- `00_入口与索引/更新记录.md`
- `03_元数据台账/rules_index.md`
- `03_元数据台账/rules_index.csv`
- `03_元数据台账/rules_index.json`
- `03_元数据台账/search.db`
- `04_托管业务专题地图/托管业务专题地图.md`
- `99_unresolved/unresolved.md`

如果 unresolved 项目已解决，应从 `unresolved.md` 移除或改为“已解决”说明，并同步到台账。涉及现行状态变化、官方来源变化或规则版本替换时，应保留清楚的更新记录，方便后续追溯。

## 附录：法规库框架和技术细节

主要目录：

- `README.md`：项目说明和 GitHub 仓库首页展示内容。
- `00_入口与索引/`：HTML 总目录和更新记录。
- `01_法规原文库/`：官方网页、PDF、DOCX 等原始材料，是最终证据源。
- `02_文本抽取库/`：从原文抽取的可检索文本。
- `03_元数据台账/`：规则编号、名称、状态、来源、路径、优先级和业务标签等结构化信息。
- `04_托管业务专题地图/`：按托管业务场景组织规则。
- `99_unresolved/`：待核验、待扩展、来源不完整或不能作为正式依据的事项。
- `tools/`：法规库构建、搜索索引构建和本地查询脚本。

本地搜索索引：

- `03_元数据台账/search.db` 是由元数据台账和文本抽取库生成的 SQLite FTS5 全文索引。
- 它只是检索加速层，不作为法规依据。
- 该文件可删除后重建，不应替代 `rules_index.*`、原文文件和官方 URL。

常用脚本：

```powershell
python tools\search_kb.py "关键词" --status 现行有效 --limit 8
python tools\build_search_index.py
python tools\build_knowledge_base.py
```

其中 `build_knowledge_base.py` 用于完整重建法规库，会刷新台账、总目录、专题地图、待核验清单并重建本地搜索索引。`build_search_index.py` 只重建搜索索引。
