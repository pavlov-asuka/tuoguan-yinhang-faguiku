# AGENTS.md

本项目是“中国商业银行托管业务法规库”，视角为商业银行开展资产托管、基金托管、理财托管、私募/资管/信托/保险资金/养老金社保年金/跨境产品托管及相关运营服务。

## 回答问题链条

1. 先做本地快速召回：优先执行 `python tools/search_kb.py "关键词" --limit 8`，满足项目本地检索要求；不凭记忆直接回答。`03_元数据台账/search.db` 只作召回加速层，不作原文依据。
2. 再核对台账口径：用 `03_元数据台账/rules_index.csv` 或 `rules_index.json` 核对法规名称、效力状态、优先级、来源 URL、本地路径和标签。遇到 `待核验`、`待扩展`、`历史失效`、`辅助资料`、`征求意见` 状态，必须说明状态，不得当作现行正式依据。
3. 按问题复杂度补查：一般解释型问题在已有必要依据后先给结论；业务场景型、跨产品或跨条线问题，优先结合 `rules_index.*` 中的产品、条线、市场标签扩展候选；必要时用 `rg` 回看 `02_文本抽取库/` 上下文。
4. 必要时联网核验：涉及现行性、最新修订、官方来源变动或精确条文时，优先使用内置 web，只取 1-2 个官方来源，并在回答中区分本地库依据与联网核验结果。
5. 控制重核验范围：不默认用 PowerShell 抓网页、下载附件或解析在线文件；只有内置 web 打不开、需要处理本地文件，或用户明确要求“完整出处/逐条核验/入库维护”时，才展开多来源交叉核验或附件解析。
6. 最后组织回答和缺口处理：尽量给出法规名称、条款位置、文件路径、效力状态和官方来源 URL；解释分析须区分“原文依据”和“分析判断”，不得表述为正式法律意见。若结论依赖但本地库未收录的法规、规则、配套文件或附件，应登记到 `00_入口与索引/unresolved.md`，标注“待月度巡检”，并在回答中说明缺口状态。

## 文件维护

- 新增法规、替换法规版本、修正元数据、补充官方来源、更新现行状态、扩展专题标签时，必须同步更新相关索引和 Markdown 文件。
- 法规库构建或文本抽取发生变化后，应同步重建 `03_元数据台账/search.db`；该文件是派生索引，可删除后用 `python tools/build_search_index.py` 重建。
- 完整重建使用：

   ```powershell
   python tools/build_knowledge_base.py
   ```

- 至少检查并按需更新：
  - `README.md`
  - `00_入口与索引/总目录.html`
  - `00_入口与索引/更新记录.md`
  - `03_元数据台账/rules_index.md`
  - `03_元数据台账/rules_index.csv`
  - `03_元数据台账/rules_index.json`
  - `03_元数据台账/search.db`
  - `00_入口与索引/unresolved.md`
- 原文文件放入 `01_法规原文库/`。
- 可检索文本放入 `02_文本抽取库/`。
- 无法确认官方正文、附件或现行状态的项目，放入 `00_入口与索引/unresolved.md`。
- 已解决的 unresolved 项目，应从 `unresolved.md` 移除或改为“已解决”说明，并同步到台账。
- 每次更改文件后，应提示用户是否需要提交 Git。
